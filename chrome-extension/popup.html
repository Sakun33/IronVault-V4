    Account

    Security model

    • Zero-knowledge: passwords decrypted locally, never sent to server.
    • Master password is never stored — only a per-session AES-GCM wrap key in chrome.storage.session.
    • Each entry's password is individually re-wrapped after unlock; only ONE password is decrypted on demand (reveal/copy/fill).
    • Auto-lock wipes the session after inactivity (1, 5, 15, or 30 min).
    • Session storage is wiped automatically when the browser closes.
    • No eval(), no inline scripts. Strict CSP: script-src 'self'; object-src 'self'.
