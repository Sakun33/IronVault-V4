🔍 Data Persistence Guide

📱 Browser-Specific Storage: Your vault data is stored locally in each browser's IndexedDB. This means data doesn't automatically sync between different browsers - this is by design for maximum privacy and security.

🚨 Issue: Data Not Syncing Across Browsers

Expected Behavior: Each browser maintains its own separate encrypted vault database. This is a feature, not a bug!

Why This Happens:

  • 🌐 Chrome has its own IndexedDB
  • 🔥 Firefox has its own IndexedDB
  • 🍎 Safari has its own IndexedDB
  • 📱 Mobile browsers each have separate databases

✅ Solution: Export/Import Between Browsers

Step 1: Export from Current Browser

  1. Open your vault in the current browser
  2. Click the "Import / Export" button
  3. Go to the "Export" tab
  4. Click "Export All Data"
  5. Download the JSON file to your computer

Step 2: Import to New Browser

  1. Open vault in the new browser
  2. Click the "Import / Export" button
  3. Go to the "Import" tab
  4. Select the downloaded JSON file
  5. Click "Import Vault"

🔒 Security Benefits

Privacy First: Browser-specific storage means your data never leaves your device and isn't transmitted over the network. Each browser acts as a separate, encrypted vault.
  • 🔐 Local Encryption: Data encrypted with AES-256
  • 🏠 Offline First: Works without internet connection
  • 🚫 No Network: Data never transmitted
  • 👤 Per-Browser: Maximum privacy isolation

💡 Best Practices

For Development/Testing

  • Use the Test Data Generator for consistent data
  • Export/Import for sharing test data between team members
  • Use the same master password across all test environments

For Production Use

  • Choose one primary browser for your vault
  • Export regularly as backup
  • Document your master password securely
  • Consider browser-specific workflows
Remember: This is a local-first, privacy-focused vault. Data persistence across browsers requires manual export/import, which is by design for maximum security and privacy.
